# Reglas de Desarrollo para Agentes en Guess It!

1. **Clean Architecture Estricta:** Respeta siempre la separación en capas (Domain, Data, Presentation). La capa de Presentación NUNCA debe comunicarse directamente con Data o Firebase; todo debe pasar a través de los UseCases del Dominio.
2. **Tipado de Fechas:** Todos los campos de fecha y hora (date and time) deben ser representados estrictamente como `String`. No utilices objetos `DateTime` nativos en las entidades ni modelos para garantizar la consistencia en el parseo y la base de datos NoSQL.
3. **Constructores y Parámetros:** Declara siempre todos los parámetros explícitamente en el código. Está estrictamente prohibido el uso de la palabra clave `super` para la inicialización abreviada en los constructores o en la herencia. Desarrolla la asignación completa de variables en el bloque de inicialización.
4. **Gestión de Estado:** Utilizamos `hydrated_bloc` y `equatable`. Todos los `State` de los BLoCs deben implementar métodos `toJson()` y `fromJson()` para persistir el estado local.
5. **Manejo de Errores:** Utiliza el paquete `fpdart`. Los UseCases deben devolver siempre `Either<Failure, Type>`. Define clases de error heredando de una clase base `Failure` en `lib/core/errors/`.
6. **Inyección de Dependencias y Rutas:** Utiliza `get_it` como Service Locator para registrar Repositorios, UseCases y BLoCs. Utiliza `go_router` para toda la navegación del sistema.
